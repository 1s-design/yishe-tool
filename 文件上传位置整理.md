# 文件上传位置整理

本文档整理了设计工具中所有可以上传文件的位置（上传到COS，然后存储到服务器）。

## 上传流程说明

所有文件上传都遵循以下流程：
1. **前端上传到COS**：使用 `uploadToCOS()` 函数（位于 `src/api/cos.ts`）
2. **存储到服务器**：将COS返回的URL及相关信息通过API保存到服务器数据库

---

## 1. 资源上传组件（主要上传入口）

**文件位置**：`src/components/design/layout/upload/index.vue`

**功能描述**：这是主要的上传入口，支持多种文件类型上传

**支持的文件类型**：
- 图片：jpg, png, svg
- 字体：ttf, woff
- 模型：glb
- PSD文件：psd

**上传逻辑**：
- **图片**：直接上传到COS → 调用 `createSticker()` 保存到贴纸库
- **字体**：生成预览缩略图 → 上传字体文件和缩略图到COS → 调用 `uploadFont()` 保存到字体库
- **模型**：生成模型预览图 → 上传模型文件和缩略图到COS → 调用 `createProductModel()` 保存到模型库
- **PSD文件**：直接上传到COS → 调用 `uploadFile()` 保存到文件库

**关键代码位置**：
- 上传函数：`uploadSingleFile()` (362-464行)
- COS上传：`uploadToCOS()` (376, 398, 402, 425, 429, 448行)
- 服务器保存：`createSticker()`, `uploadFont()`, `createProductModel()`, `uploadFile()`

## 核心API函数

### COS上传函数
**位置**：`src/api/cos.ts`
```typescript
export async function uploadToCOS({
    file,
    key = new Date().getTime() + '_1s_' + file.name,
})
```
- 功能：上传文件到腾讯云COS
- 返回：`{ url: string, key: string }`

### 服务器保存API
**位置**：`src/api/index.ts`

- `createSticker()` - 创建贴纸/图片素材
- `uploadFont()` - 上传字体
- `uploadFile()` - 上传通用文件
- `createProductModel()` - 创建产品模型
- `createCustomModelApi()` - 创建自定义模型
- `updateCustomModel()` - 更新自定义模型
- `createDraft()` - 保存到草稿箱
- `updateUserInfo()` - 更新用户信息
- `updateFontTemplate()` - 更新字体模板

## 总结

### 上传文件类型统计
1. **图片**：jpg, png, svg, webp
2. **字体**：ttf, woff, otf
3. **模型**：glb, gltf
4. **视频**：webm
5. **设计文件**：psd

### 上传场景分类
- **素材上传**：图片、字体、模型、PSD文件
- **编辑后上传**：图片编辑器、画布内容、文字贴纸
- **缩略图上传**：字体预览图、模型预览图
- **用户数据上传**：头像
- **草稿箱上传**：截图、录制视频、多角度图片
- **管理功能上传**：产品模型

### 统一的上传模式
所有上传都遵循：**本地文件 → COS存储 → 服务器数据库** 的流程，确保文件安全存储和元数据管理。
